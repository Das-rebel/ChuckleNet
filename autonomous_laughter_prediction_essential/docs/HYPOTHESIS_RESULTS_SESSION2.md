## H2.5: MULTI-WORD SPAN VERIFICATION — CONFIRMED BUT MISLEADING

### Results
- 198 laughter spans analyzed across 15 videos
- **100% of spans are multi-word** (2+ consecutive label=1 words)
- Average span length: **19.4 words**
- 74.7% of spans are >11 words

### Interpretation
The "100% multi-word" result is NOT evidence that laughter naturally spans multiple words. It's an **artifact of the alignment algorithm**: when a [laughter] marker appears in the VTT, ALL words within the laughter window (±5s) are labeled as 1. This means our weak labels inherently produce multi-word spans by design, not by nature.

**Implication for span reformulation thesis:** The >70% multi-word claim is TRUE for our data, but it doesn't prove spans are a better cognitive model of laughter — it proves our labeling method creates spans. This weakens the "human annotation" argument for spans (H2.2).

## H4.5: DATA SPLIT LEAKAGE — MINOR

### Results
| Split Method | F1 | Gap |
|-------------|:---:|:---:|
| Random (cross-video) | 0.218 | — |
| Per-Show (same video in one split) | 0.199 | **+1.9%** |

### Interpretation
- **1.9% gap** → minor leakage from random splits
- Below the 3% threshold for "significant leakage"
- BUT: the gap exists. When the same comedian's style appears in both train and val, the model overfits slightly
- **Recommendation:** Use per-show splits for final paper results. Report random-split as an upper bound.

## 🔥 H4.4: BIOSEMOTIC FEATURE LEAKAGE — CONFIRMED (CRITICAL)

### The Experiment
Trained LogisticRegression on **ONLY biosemotic features** (10 numeric dimensions: duchenne_*, incongruity_*, tom_*). **NO text. NO XLM-R. NO words.**

### Results
| Feature Set | F1 |
|-------------|:---:|
| **ALL biosemotic features (no text)** | **0.8289** |
| Duchenne only | 0.7989 |
| Incongruity only | 0.5761 |
| Theory of Mind only | 0.5010 |

### The Smoking Gun
```
Biosemotic features ALONE:        F1 = 0.8289
Our XLM-R model (with text):      F1 = 0.819
Our XLM-R model (text only):      F1 ≈ 0.816
TF-IDF on text alone:             F1 ≈ 0.22
```

**The biosemotic features contain essentially ALL the label information.** They were generated by an LLM that had access to the labels and encoded them into the feature values. 

### What This Means

1. **All biosemotic results are invalid** — the features are a label proxy, not a genuine signal
2. **The +0.003 F1 improvement was meaningless** — XLM-R already saw the labels through the features
3. **Our XLM-R performance WITHOUT features is ~0.816** — still very strong
4. **The LLM that generated these features committed data leakage** — it was given the task "generate features for laughter" and it encoded the answer into the features

### Verdict
**🔴 CRITICAL FINDING — This is a publishable negative result.** It proves that synthetic feature generation by LLMs can introduce label leakage that invalidates ablation studies. The fact that 10 hand-crafted numeric features achieve F1=0.829 without seeing any text is impossible under normal circumstances — the features are cheating.
